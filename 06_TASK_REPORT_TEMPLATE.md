# 06_TASK_REPORT_TEMPLATE.md

## Назначение

Этот файл задает обязательный формат отчета после выполнения задачи.

Агент обязан использовать этот шаблон после любого изменения кода, архитектуры, файлов, настроек, инструкций или технического плана.

---

```markdown

- `path/to/file`

- ...

- `npm run lint` — результат

## Как запустить

...

1. ...
2. ...
3. ...

Ожидаемый результат:
- ...

## Что не удалось проверить

- ...

Причина:
- ...

## Риски

- ...

## Следующий безопасный шаг

- ...
```

---

## Запрещенные формулировки

Нельзя писать:

- “всё готово”, если проверки не выполнены;
- “должно работать”, если это не проверено;
- “ошибок нет”, если сборка не запускалась;
- “платежи подключены”, если нет webhook;
- “безопасно”, если не проверены права доступа;
- “готово к production”, если нет проверки сборки, безопасности и основных сценариев.

---

## Правильные формулировки

Если проверено:

```text
Сборка прошла. Основной сценарий проверен. Критических ошибок не обнаружено.
```

Если не проверено:

```text
Код реализован, но сборка не запускалась из-за отсутствия окружения. Для подтверждения нужно выполнить ...
```

Если частично проверено:

```text
Проверена структура и типы. Runtime-сценарий нужно дополнительно проверить вручную.
```
